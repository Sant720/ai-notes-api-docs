# Retrieve all notes

GET https://{url}/notes 

Retrieve all notes belonging to the authenticated user.

## Overview
Returns a paginated list of notes owned by the authenticated account.  
Requires a valid Bearer token.

**Base URL**
```
https://389d0769-9386-445d-bca6-1a7753678ff2.mock.pstmn.io
```


## Authentication
This endpoint requires an `Authorization` header with a valid Bearer token:

```
Authorization: Bearer demo-valid
Accept: application/json
```


## Example requests

```bash
curl 'https://389d0769-9386-445d-bca6-1a7753678ff2.mock.pstmn.io/notes' --header 'Authorization: Bearer demo-valid' --header 'Accept: application/json'
```

## Possible responses

| Status | Meaning | Description |
|--| --|--|
| 200 | OK | Notes retrieved successfully |
| 401 | Unauthorized | Missing or invalid token |
| 403 | Forbidden | Token valid but lacks permission |
| 404 | Not Found | No notes for this user |
| 429 | Too Many Requests | Rate-limit exceeded |
| 500 | Internal Server Error | Unexpected backend issue |
| 503 | Service Unavailable | Temporary maintenance |


## Headers

| Name | Type | Required | Description |
|--|--|--|--|
| `Authorization` | string | ✅ | Bearer token for authentication |
| `Accept` | string | ✅ | Expected response type (`application/json`) |


## Notes
- `GET /notes` is scoped to the authenticated user — no `user_id` parameter is required.  
- Rate-limit headers (`X-RateLimit-*`, `Retry-After`) are returned when limits are hit.  
- You can use the Postman mock server URL directly to test the responses.
